1. Prerequisites

WooCommerce store, administrator access, approved marketplace account, public image URLs for real image sync, and API/OAuth credentials where required.

2
2. Installation

Install the Lite plugin from WordPress.org when available, or upload the PRO ZIP through WordPress Admin → Plugins → Add New → Upload Plugin. Activate WooCommerce first.

3
3. API and OAuth configuration

Enter client ID/secret or marketplace credentials in plugin settings. For OLX.bg use the exact OAuth callback URL shown in the plugin settings before connecting the account.

4
4. Category and attribute mapping

Map each WooCommerce category to the remote marketplace category. Add required attributes such as condition/state and category-specific code=value attributes before publishing.

5
5. Images and moderation

Localhost and private URLs cannot be fetched by marketplaces. Use a public site/CDN for product images and avoid fake/test listing titles in live sync.

6
6. First test sync

Start with read-only/API validation, then run one controlled product sync. Confirm remote advert ID, status, URL and WooCommerce metadata before enabling bulk or automated sync.

OLX.ba

OLX.ba notes

Preserve existing listing IDs, category mappings and sync flags. Review required category attributes before create/update operations.

OLX.bg

OLX.bg notes

Use Partner API v2, OAuth token storage, state=new for common goods categories, and update existing adverts by advert ID to avoid duplicates.
